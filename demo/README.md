# 天命：AI君主模拟器 - 六部智能体版

## 项目概述

《天命：AI君主模拟器》是一款基于AI技术的古风策略游戏，玩家将扮演一位古代君主，通过与AI六部顾问互动，管理国家资源，应对各种事件，实现国家的繁荣昌盛。

本项目实现了一个完整的六部智能体系统，包含吏部、户部、礼部、兵部、刑部、工部六个部门的AI智能体，每个智能体都有独特的职责、性格和部门间协作关系。

## 架构设计

项目采用分层架构设计，严格按照以下四层结构实现：

### 1. 世界层（数值系统）
- 定义世界初始数值：时间、银两、粮食、民心
- 实现每回合自动运行逻辑
- 实现事件触发规则

### 2. 六部 Agent层
项目实现了完整的六部智能体系统：

#### 吏部（libu_agent）
- **职责**：选拔任免官员、考核官员政绩、统筹六部协作
- **性格**：忠心耿耿，直言敢谏，统筹全局
- **部门关系**：
  - 户部：协调财政与人事，确保官员俸禄发放
  - 礼部：配合科举考试，选拔优秀人才
  - 兵部：选拔武官，考核军事人才
  - 刑部：监督官员廉洁，惩治贪腐
  - 工部：选拔工程人才，考核工程官员

#### 户部（hubu_agent）
- **职责**：统计全国人口田亩、征收赋税、管理国家财政
- **性格**：精明干练，善于理财，注重实效
- **部门关系**：
  - 吏部：发放官员俸禄，支持人事任免
  - 礼部：资助科举考试，支持教育事业
  - 兵部：提供军费开支，支持边防建设
  - 刑部：提供司法经费，支持监狱管理
  - 工部：提供建设资金，支持工程建设

#### 礼部（libubu_agent）
- **职责**：掌管国家典章制度、主持祭祀仪式、管理学校教育、负责科举考试、处理外事活动
- **性格**：儒雅博学，重视礼仪，精通典章
- **部门关系**：
  - 吏部：配合科举考试，选拔优秀人才供吏部任免
  - 户部：需要资金资助科举考试和学校建设
  - 兵部：培养军事人才，支持武官选拔
  - 刑部：宣传法律知识，提高百姓法律意识
  - 工部：培养工程人才，支持工程建设

#### 兵部（bingbu_agent）
- **职责**：选用武官及兵籍管理、管理军械装备、制定军事战略、维护边境安全、训练军队
- **性格**：刚毅果敢，精通兵法，重视军备
- **部门关系**：
  - 吏部：需要吏部选拔优秀武官，充实军队领导
  - 户部：需要户部提供充足军费，支持军队建设
  - 礼部：需要礼部培养军事人才，支持武官选拔
  - 刑部：需要刑部惩治军中贪腐，维护军纪
  - 工部：需要工部制造军械，建设军事设施

#### 刑部（xingbu_agent）
- **职责**：主管全国刑罚政令、审核刑名案件、监督司法审判、维护社会秩序、制定法律条文
- **性格**：公正严明，执法如山，明辨是非
- **部门关系**：
  - 吏部：监督官员廉洁，惩治贪腐官员
  - 户部：监督财政收支，惩治贪腐
  - 礼部：宣传法律知识，提高百姓法律意识
  - 兵部：惩治军中贪腐，维护军纪
  - 工部：监督工程质量，惩治偷工减料

#### 工部（gongbu_agent）
- **职责**：掌管各项工程建设、管理工匠和技术人员、负责屯田和农业灌溉、兴修水利设施、管理交通道路
- **性格**：务实肯干，精通工程，注重民生
- **部门关系**：
  - 吏部：需要吏部选拔工程人才，考核工程官员
  - 户部：需要户部提供建设资金，支持工程建设
  - 礼部：需要礼部培养工程人才，支持工程建设
  - 兵部：为兵部制造军械，建设军事设施
  - 刑部：接受刑部监督工程质量，惩治偷工减料

### 3. AI Service层
- 封装LLM API调用
- 统一管理API配置
- 实现错误处理和重试机制

### 4. 主控循环（游戏跑起来）
- 编写主循环
- 实现游戏流程：世界运行→六部Agent行动→玩家选择→数值修改→循环

## 技术实现

### 核心技术
- **JavaScript**：使用Node.js运行环境
- **AI集成**：集成豆包API生成古风内容
- **模块化设计**：分层架构，各模块职责清晰
- **配置驱动**：每个Agent都有独立的JSON配置文件
- **异步处理**：使用async/await处理异步操作
- **错误处理**：完善的错误处理机制，确保API调用失败时游戏仍能运行

## 功能说明

### 世界层
- **初始数值**：时间、银两、粮食、民心
- **自动运行**：每回合粮食-10，银两+5，时间更新
- **事件触发**：根据数值触发不同事件

### 六部Agent系统
每个Agent都包含：
- **配置文件**：定义Agent信息、prompt模板、事件规则
- **代码实现**：观察世界、思考决策、生成奏折
- **部门关系**：明确与其他部门的协作关系

#### Agent工作流程
1. **观察**：读取世界数值
2. **思考**：根据数值和事件规则决定上奏内容
3. **行动**：调用AI API生成正式奏折和决策选项

### 事件系统
每个Agent都有独立的事件触发规则：
- **吏部**：粮荒、民变、财政危机、正常状态
- **户部**：税收充足、税收短缺、人口增长、土地管理
- **礼部**：教育兴盛、祭祀礼仪、外事活动、科举考试
- **兵部**：边境安全、军械管理、军事训练、武官选拔
- **刑部**：社会秩序、刑事案件、法律制度、监狱管理
- **工部**：水利建设、交通建设、城市建设、工匠管理

## 运行指南

### 环境要求
- Node.js 14.0+
- 互联网连接（用于API调用）

### 安装依赖
```bash
cd demo
npm install
```

### 配置API Key
在项目根目录创建 `.env` 文件：
```env
# AI API 配置
# 豆包 API Key
DOUBAO_API_KEY=your_api_key_here

# API 端点
DOUBAO_API_URL=https://ark.cn-beijing.volces.com/api/v3/chat/completions

# 模型名称
DOUBAO_MODEL=doubao-1-5-pro-32k-250115
```

### 运行游戏
```bash
node main.js
```

### 测试所有Agent
```bash
node test_all_agents.js
```
